# Chat IM UX Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the Sparkbox Mobile `Chats` tab into a user-friendly messaging surface with a real conversation list, clearer group/private chat identity, and more readable conversation detail without changing the existing shell-state architecture.

**Architecture:** Keep the current `App.tsx` shell state machine and rebuild the `Chats` tab inside that boundary. Push list/detail copy and state rules into `src/spaceShell.ts` and `src/appShell.ts`, then update `App.tsx` to render richer list rows, a messenger-style detail header, and grouped message rhythm driven by existing session summary/detail data.

**Tech Stack:** React Native, TypeScript, Vitest, existing household/chat APIs in `src/householdApi.ts`
